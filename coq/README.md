# A Two-Level Linear Dependent Type Theory — Rocq formalization

Mechanized metatheory for Two-Level Linear dependent type theory (TLL).

## Build

Dependencies (via opam):
[`rocq-prover`](https://rocq-prover.org/),
[`coq-mathcomp-ssreflect`](https://math-comp.github.io/),
[`coq-extructures`](https://github.com/arthuraa/extructures),
[`coq-deriving`](https://github.com/arthuraa/deriving).
A vendored copy of [Autosubst](https://www.ps.uni-saarland.de/autosubst/) lives
under [autosubst/](autosubst/), so no separate Autosubst install is needed.

```
make -j
```

`_CoqProject` lists every file in build order. `Makefile` is generated by
`coq_makefile` and reads `_CoqProject` via `Makefile.conf`.

## Directory layout

- [autosubst/](autosubst/) — vendored Autosubst library (substitution
  infrastructure used by every `term` definition).
- [theories/](theories/) — all of TLL's metatheory. Files are named
  `<fragment>_<topic>.v`; see below.

## Fragments

Each fragment has its own term/typing definitions in `<fragment>_ast.v` (where
present) and `<fragment>_type.v`. A standard set of topic files then proves the
usual metatheoretic lemmas about that fragment.

| Fragment   | Prefix                | What it is                                                                                                         |
| ---------- | --------------------- | ------------------------------------------------------------------------------------------------------------------ |
| Preamble   | `ARS`, `AutosubstSsr` | Abstract reduction systems + ssreflect glue for Autosubst.                                                         |
| MLTT       | `mltt_`               | A plain Martin-Löf type theory, used as the strong-normalization target for TLL's logical level via a translation. |
| TLL syntax | `tll_ast`             | Shared term language for both TLL fragments.                                                                       |
| Logical    | `logical_`            | TLL's logical level (`Γ ⊢ m : A`) .                                                                                |
| Program    | `program_`            | TLL's program level (`Γ ; Δ ⊢ m : A`) .                                                                            |
| Erasure    | `erasure_`            | Erasure relation for program terms, with its own reduction.                                                        |
| Heap       | `heap_`               | A small abstract machine: heap-allocated values, resolution, pointer-level reduction.                              |

## Topic suffixes

Within a fragment, suffixes follow a fixed convention:

| Suffix    | Contents                                                                  |
| --------- | ------------------------------------------------------------------------- |
| `_ast`    | Syntax (terms, sorts).                                                    |
| `_ctx`    | Contexts, well-formedness, lookup/`has`.                                  |
| `_step`   | Single-step and multi-step reduction.                                     |
| `_conf`   | Confluence / Church–Rosser.                                               |
| `_type`   | Typing judgment.                                                          |
| `_weak`   | Weakening / renaming.                                                     |
| `_subst`  | Substitution lemma.                                                       |
| `_inv`    | Inversion lemmas for the typing judgment.                                 |
| `_unique` | Sort/type uniqueness (logical fragment).                                  |
| `_valid`  | Validity (types of well-typed terms are well-typed).                      |
| `_sr`     | Subject reduction.                                                        |
| `_sn`     | Strong normalization (logical fragment, via the MLTT translation).        |
| `_prg`    | Progress.                                                                 |
| `_defs`   | Heap-specific definitions (heap as a finite map).                         |
| `_res`    | Heap resolution: relating closed terms to heap-allocated representations. |

## Headline theorems

- Logical confluence — [logical_conf.v](theories/logical_conf.v)
- Logical subject reduction — `logical_sr` in [logical_sr.v](theories/logical_sr.v)

- Heap progress — `heap_prg` in [heap_prg.v](theories/heap_prg.v)
